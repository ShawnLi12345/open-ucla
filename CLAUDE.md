## Working Principle
0. There will always be two branches: main and development, if development does not exist, create it
1. When working on a feature/fix, create a new git branch based from development branch. 
 - Naming Convention: dev/(branch name)
2. When done, create a pr on github cli to the development branch. 
3. Never push to main branch.
